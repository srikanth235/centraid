# HTTP API

This page documents the HTTP surface served by `@centraid/app-engine` (`Runtime.handle`), plus the two store route groups (`/_centraid-conversations`, `/_centraid-user`) that the host mounts alongside it. Same surface on the local desktop gateway and the remote OpenClaw plugin.

> App **publishing / versioning** (uploading code, version history, activation) is **not** part of this surface — code lives in the gateway's git store (issue #137), and that publish API lives in `@centraid/gateway`. The routes below are the runtime/data/chat surface only.

## Registry

| Method   | Path                   | Purpose                                               |
| -------- | ---------------------- | ----------------------------------------------------- |
| `GET`    | `/centraid/_apps`      | List registered apps → `[{ id, path, registeredAt }]` |
| `DELETE` | `/centraid/_apps/<id>` | Deregister an app (and clean up its per-app dir)      |

The registry persists at `<appsDir>/_registry.json` as `{ apps: [{ id, path, registeredAt }] }`, written via tmp-file + atomic `rename`. There is no `upload` / `activate` / `versions` route here.

## Cloud-panel data routes

Read/inspect an app's `data.sqlite` and run ad-hoc SQL (the desktop's row browser, SQL editor, logs):

| Method | Path                                | Purpose                                                 |
| ------ | ----------------------------------- | ------------------------------------------------------- |
| `GET`  | `/centraid/_apps/<id>/schema`       | Live schema (tables, columns, indexes, views)           |
| `GET`  | `/centraid/_apps/<id>/data/<table>` | Table rows — `?limit=&offset=`                          |
| `POST` | `/centraid/_apps/<id>/query`        | Run SQL; a successful write emits a `'external'` change |
| `GET`  | `/centraid/_apps/<id>/logs`         | Read the app's `logs.jsonl`                             |

## Three-tool dispatcher

| Method | Path                                | Purpose                |
| ------ | ----------------------------------- | ---------------------- |
| `POST` | `/centraid/_tool/centraid_describe` | Manifest introspection |
| `POST` | `/centraid/_tool/centraid_read`     | Invoke a query         |
| `POST` | `/centraid/_tool/centraid_write`    | Invoke an action       |

Results are MCP-shaped (`{ isError, content, structuredContent }`). The shim maps `structuredContent.code` to an HTTP status: `UNKNOWN_APP`/`UNKNOWN_ACTION`/`UNKNOWN_QUERY` → 404, `WRONG_KIND`/`INVALID_INPUT` → 400, `NO_ACTIVE_VERSION` → 503, `INVALID_MANIFEST`/`HANDLER_ERROR` → 500. Full request/response shapes: [Three-tool dispatcher](/reference/three-tool-dispatcher).

## Runner preflight

| Method | Path                            | Purpose                                          |
| ------ | ------------------------------- | ------------------------------------------------ |
| `GET`  | `/centraid/_turn/runner-status` | Adapter readiness + model catalog (`?refresh=1`) |

Returns `{ kind, ok, version?, models?, modelsStatus?, … }` so the chat panel can show a setup screen instead of failing per-turn. Returns `{ kind: 'none', ok: false }` when no runner-status reporter is wired.

## Per-app surface

| Method | Path                      | Purpose                                      |
| ------ | ------------------------- | -------------------------------------------- |
| `GET`  | `/centraid/<id>/`         | Serves `index.html` from the live code dir   |
| `GET`  | `/centraid/<id>/<file>`   | Static asset (extension allowlist)           |
| `GET`  | `/centraid/<id>/_changes` | SSE stream of `data.sqlite` mutations        |
| `POST` | `/centraid/<id>/_turn`    | Send a chat turn → SSE stream of turn events |

A `/centraid/_draft/<sessionId>/<id>/…` prefix (issue #141) reuses these same routes against a draft worktree's code with the app's live data.

### Static-asset extension allowlist

```
.html .htm .css .js .mjs .json .svg .png .jpg .jpeg .webp .gif .ico
.woff .woff2 .ttf .otf .map
```

Anything else returns `null` (→ 404). Reserved names `data.sqlite` and `_registry.json`, and the `queries/` and `actions/` directories, are never served. Note `app.json` **is** served — the desktop settings popover fetches it to read the manifest's `knobs` array, and the manifest is the (non-secret) agent-facing tool contract.

### `GET /centraid/<id>/_changes` — change stream

Server-sent events. One `change` event per successful write, naming the touched tables:

```
event: change
data: {"tables":["todos"],"ts":1715812345678,"source":"handler"}
```

Heartbeat comment every 30s. Full payload (`source`, `toolCallId`, `turnId`): [Change stream](/concepts/change-stream).

### `POST /centraid/<id>/_turn` — chat turn

Request body:

```json
{
  "conversationId": "<conversation id>",
  "message": "user input",
  "model": "<runner model id>",
  "thinking": "...",
  "idempotencyKey": "...",
  "attachments": [{ "hash": "<sha256>", "mime": "image/png", "filename": "shot.png" }]
}
```

`conversationId` and `message` are required; the rest are optional. `attachments` reference bytes already uploaded to `/_centraid-conversations/apps/<id>/blobs`. Response is an SSE stream of `TurnStreamEvent`s (`assistant.delta`, `tool.start`, `tool.result`, `final`, `usage`, `error`, `aborted`, `webhooks`, …) ending with `event: end`. Without a configured runner the route 503s with `no_conversation_runner`. Full event list: [Chat](/concepts/chat).

## Conversation store — `/_centraid-conversations`

Mounted by the host against `runtime.conversationHistoryStore` (the chat transcript + session list). App-scoped; per-user scoping enforced inside the store.

| Method   | Path                                                | Purpose                                                    |
| -------- | --------------------------------------------------- | ---------------------------------------------------------- |
| `GET`    | `/_centraid-conversations/apps/<id>/sessions`       | List → `{ sessions: [...] }`                               |
| `POST`   | `/_centraid-conversations/apps/<id>/sessions`       | Create — body `{ title? }`                                 |
| `GET`    | `/_centraid-conversations/apps/<id>/sessions/<sid>` | Load with reconstructed transcript                         |
| `PATCH`  | `/_centraid-conversations/apps/<id>/sessions/<sid>` | Rename — body `{ title }`                                  |
| `DELETE` | `/_centraid-conversations/apps/<id>/sessions/<sid>` | Delete (cascades + blob GC)                                |
| `POST`   | `/_centraid-conversations/apps/<id>/blobs`          | Upload attachment bytes → `{ hash, sizeBytes, mime, url }` |
| `GET`    | `/_centraid-conversations/apps/<id>/blobs/<hash>`   | Download attachment bytes                                  |

Blob uploads cap at 25 MB; the hash is the sha256 content key. Downloads serve `cache-control: immutable`.

## User prefs — `/_centraid-user`

Mounted against `runtime.userStore` (gateway-wide user prefs, single-user today). Used by the desktop to read/write global settings; see `makeUserStoreRouteHandler`.

## Cross-cutting

### Security headers (static assets)

Static responses carry:

- `Content-Security-Policy: default-src 'self'; script-src 'self'[ 'nonce-…'] ; style-src 'self' 'unsafe-inline'; img-src 'self' data:; object-src 'none'; base-uri 'self'; frame-ancestors 'self'`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: no-referrer`

`index.html` gets a per-response `nonce` so the live-settings bridge's inline `<script>` is allowed without opening `script-src` to all inline scripts.

### Body limit

`MAX_BODY_BYTES` = 1 MiB on the turn route and the tool shim. The conversation store's blob upload has its own 25 MB cap.

### Reserved ids

App ids starting with `_` are reserved (`_apps`, `_tool`, `_turn`, `_changes`); they also can't contain `/`, `..`, or be empty. Attempts to register one throw a `RegistryError`.
