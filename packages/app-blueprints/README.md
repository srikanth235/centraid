# @centraid/app-blueprints

Bundled, pre-built Centraid apps that the desktop gallery offers as **clone and deploy** starting points. Templates are split by kind: full UI apps live under `apps/<id>/` and automation apps under `automations/<id>/`. Each is a fully-formed app — identical in shape to an app a user authors themselves (`index.html`, `app.css`, `app.js`, `queries/`, `actions/`, `migrations/`, `app.json`).

## Layout

```
packages/app-blueprints/
  src/
    index.ts        — listTemplates(), appTemplatesDir, types
    types.ts        — TemplateMeta, TemplateManifest
  index.json        — manifest source (TemplateManifest)
  manifest.json     — generated by scripts/build-manifest.mjs
  apps/<id>/        — one folder per UI-app template (kind: 'app')
  automations/<id>/ — one folder per automation template (kind: 'automation')
```

The kind-segment directory (`apps/` vs `automations/`) is **derived from `kind`**, not stored separately — the manifest build, the on-disk resolver, and the remote fetcher all compute it the same way (`templateKindDir`). The bundled tree doubles as the remote tree, so a GitHub-raw URL serving `manifest.json` + `<apps|automations>/<id>/<file>` works as a remote template source unchanged.

## Adding a template

1. Drop the app folder at `apps/<id>/` (or `automations/<id>/` for an automation) with the standard layout (`index.html`, `app.css`, `app.js`, `queries/`, `actions/`, `migrations/`, `package.json`, `tsconfig.json`, `app.json`).
2. **Pre-build the handlers**: commit the compiled `.js` files alongside `.ts` so cloning is "click and deploy" — no `bun install` needed on first use.
3. Add an entry to `index.json` with `id`, `name`, `desc`, `colorKey`, `iconKey`, `version` (and `kind: 'automation'` for automation templates — UI apps omit `kind`).

The `version` field follows semver. Bump it when the template's source changes.

## Cloning at runtime

The gateway owns the template catalog: it serves display metadata at `GET /centraid/_templates` (resolved from `listTemplates()`), and clones a chosen template into a draft git-store session over HTTP via `@centraid/gateway`'s lifecycle routes. The publish step flips the draft live.
