## What changed

**Core pack re-pinned to upstream SHA `2a18ad43`.** Lockfile upserted via `packverb lock-add` — the prior `0.3 / 18098b66` entry was replaced atomically by pack id (no manual lockfile edit). The lockfile now records `governance-kit/core` at `version: 0.3.1`, ref `gh:Duaility/governance-kit/packs/core@main`, sha `2a18ad43599ba90ac14175ad34d72c4013f485ce`, `min_governance_kit: 0.3`.

**13 installed directive folders overwritten with 0.3.1 content.** Re-fetched the pack from cache, then iterated every directive currently listed in `packs.lock` (agent-steering-accounting, agent-token-accounting, commit-issue-receipt-match, commit-message-format, doc-freshness, issue-templates, issues-tracked, no-broken-internal-doc-links, no-orphan-todos, receipt-per-issue, repo-hygiene, required-docs, secrets-hygiene) and called `install_directive_folder` to replace each in place. `evals/` and `install-assets/` excluded by the helper (target-repo install policy). Substantive diffs land in the next two paragraphs; the rest are quote-style normalizations.

**Substantive directive changes.** Two upstream bugfix PRs land:
- `agent-token-accounting` ([Duaility/governance-kit#138](https://github.com/Duaility/governance-kit/pull/138)) — `lib/trailers.py` now exposes a `validate-blocks` entry point that splits the commit body into trailer-only paragraphs and validates each `(Cost-Key, Token-*, Cost-USD)` block against its own `COSTS.md` row. The previous global last-wins parser kept only the trailing sub-commit's trailers and silently skipped the rest, leaving every other folded sub-commit's ledger row unverified. `check.sh` also adds a Mode B HEAD fallback: when the merge-base walk finds no new work (typical for squash-merge commits landing on `main` via GitHub's server, which bypass the local commit-msg hook), the directive now validates HEAD's trailer blocks on its own. `constitution.md` updated to document both behaviors.
- `agent-steering-accounting` ([Duaility/governance-kit#137](https://github.com/Duaility/governance-kit/pull/137)) — `lib/trailers.py` adds `extract_summed_trailers`, which sums `Steer-Count` / `Steer-Types` / `Steer-Tiers` across every occurrence in the body rather than picking the last. Same squash-merge concern: a squashed body concatenates one trailer block per folded sub-commit, and the cumulative `STEERING.md` diff has to agree with the trailer math by construction.

**Cosmetic directive changes.** `commit-message-format/directive.yaml`, `doc-freshness/directive.yaml`, and `no-orphan-todos/directive.yaml` each had their `summary:` field re-quoted from single to double quotes. No behavior change.

**Hook dispatchers regenerated by hooks.sh.** All five `.githooks/*` hooks re-emitted via `generate_hooks_for_strategy githooks 0.3`. Marker line bumped to `generated=2026-05-26`; bodies are byte-equivalent to the 0.3 generation except where the lint-guard preservation below applies.

**Lint-guard block from #19 preserved in pre-commit.** Same drill as the 0.3 upgrade: the regenerated `pre-commit` dropped the hand-added `scripts/lint-staged.sh` shim block. Re-injected at the same anchor (immediately after the `GOVERNANCE_DIR=…` assignment, before `directive_field()`). The block itself anticipates this regeneration and points at `scripts/lint-staged.sh` as the canonical implementation, so the re-injection follows the block's own playbook.

**`workflows-hardened` directive skipped intentionally.** Upstream 0.3.1 added a new `workflows-hardened` directive and added it to the `minimal` preset. `pack update` does not auto-install new directives — promotion-into-preset is not authority to install. User explicitly opted to skip it for this upgrade; can be installed later with `governance directive add --pack governance-kit/core workflows-hardened` (or `pack remove` then `pack add` against the preset).
