Centraid · personal software

Software for a market of one.

Most software is built for everyone, which is why none of it quite fits you. Centraid is a personal app builder: describe the app your life needs, an agent writes it, a gateway on your own machine runs it, and it appears on your desktop and phone — a folder of plain files and one SQLite database.

How it worksin five chapters

01 the ask

You describe it. An agent builds it.

There is no project to scaffold and nothing to configure. You say what you want in a sentence. The builder agent writes the app on a draft branch, you poke at a live preview, and Publish flips it onto every device you own.

Changed your mind next month? Say that too. The same conversation that built the app can rewrite it.

Drafts are git branches in a local store — publish is a fast-forward, and every version stays. How apps are put together →

02 the artifact

Small enough to read in an evening.

An app is a folder: an HTML page, plain JavaScript handlers, SQL migrations, one manifest. Its entire life's data is one SQLite file on your disk. No node_modules, no bundler, no deploy pipeline — the runtime loads the files as they are.

When something feels off, you don't file a ticket. You open the folder and read it. All of it.

Handlers are pure functions over the app's own database. Type hints come from JSDoc, not a compile step. Queries and actions →

03 the runtime

Chat isn't a feature. It's the runtime.

Every app declares what it can be asked (queries) and what it can be told (actions). The buttons in the UI call that catalog — and so does the agent. Saying it and tapping it are the same code path, so talking to an app is full-fidelity use, not a gimmick bolted on top.

Three generic tools fan out to every app's declared handlers — agents, automations, and the UI share one calling convention. The dispatcher →

chat · patchlive
you watered the tomatoes and the chillies — and I spotted aphids on the chillies again
patch ⚙ write → water { bed: "tomatoes" } ⚙ write → water { bed: "chillies", note: "aphids" } Both logged for today, chillies flagged. You last saw aphids there twelve days ago — that's twice this month.

04 the long game

Automations are conversations that don't end.

A morning briefing, an email triage, a nag to water the chillies — each one is an agent on a schedule or a data trigger, working over your apps while you're away. Every run keeps a durable transcript.

So when you wonder why it did what it did, you don't dig through logs. You read the conversation back.

Cron, webhook, or a change in your data — ten templates to start from. Automations →

05 the vault

Your data can say no.

Apps don't own your records — they borrow them. Your health readings, your people, your notes live in one vault on your machine. Each app asks for exactly the scopes it needs, and you sign the grant.

Revoke it, and the app goes dark — your data stays. Try asking that of a SaaS account.

Where it runsone gateway · three homes

It runs where your life runs.

local-first: per-app SQLite on your machine, a single bearer token, and nothing leaves your devices unless you point it somewhere.

home 1

Desktop shell

The gateway embedded in an app on your Mac, Linux, or Windows machine. Open it like anything else.

home 2

Always-on daemon

centraid serve on a spare machine or home server — your apps and automations keep working with the laptop closed.

home 3

Phone in your pocket

The mobile companion is a thin client onto any gateway you run. Same apps, same data, no sync service.

→ get started

Sixty seconds to your first app.

# macOS · Linux · Windows — shell, gateway, and CLI, wired up
$ curl -fsSL https://centraid.dev/install.sh | bash
# needs Bun ≥ 1.3 and Node ≥ 24
$ git clone https://github.com/srikanth235/centraid
$ cd centraid && bun install
$ bun run dev:desktop

Then open the shell with centraid open, or run a shareable gateway with centraid serve.

Five minutes in, you can clone a template, watch a write round-trip through the dispatcher, and ask your new app a question. The quickstart walks the whole loop.
